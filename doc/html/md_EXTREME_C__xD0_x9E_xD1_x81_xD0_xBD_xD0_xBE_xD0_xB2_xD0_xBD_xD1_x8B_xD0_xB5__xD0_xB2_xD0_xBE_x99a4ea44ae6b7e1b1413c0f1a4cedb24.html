🚨 Типичные ошибки новичков при работе с макросами

1. Отсутствие скобок в функциональных макросах

Самая частая логическая ошибка в Си, ломающая приоритет математических операций. ❌ Плохо:

#define MULTIPLY(a, b) a * b
int x = MULTIPLY(2 + 3, 4); // Развернется в: 2 + 3 * 4 = 14 (Ожидалось: 20)

✅ Как исправить:

#define MULTIPLY(a, b) ((a) * (b))
int x = MULTIPLY(2 + 3, 4); // Развернется в: ((2 + 3) * (4)) = 20

2. Побочные эффекты (Инкременты в аргументах)

Если передать в макрос переменную с инкрементом (++), операция выполнится несколько раз, так как текст подставляется напрямую. ❌ Плохо:

#define SQUARE(x) ((x) * (x))
int i = 3;
int result = SQUARE(++i); // Развернется в: ((++i) * (++i)) -> i увеличится дважды!

✅ Как исправить: Для сложных вычислений в современном Си (начиная с C99) лучше использовать статические инлайновые функции: static inline int square(int x) { return x * x; }.

3. Лишняя точка с запятой <tt>;</tt> в конце <tt>#define</tt>

Препроцессор бездумно подставляет всё, что вы написали, включая точку с запятой, ломая синтаксис. ❌ Плохо:

#define MAX_VALUE 100; // Обратите внимание на ';' в конце
int arr[MAX_VALUE]; // Развернется в ошибку: int arr[100;];

✅ Как исправить: Никогда не ставьте ; в конце директив препроцессора.

4. Попытка проверить значение через <tt>#ifdef</tt> вместо <tt>#if</tt>

#ifdef проверяет только факт существования макроса, а не то, что внутри него. ❌ Плохо:

#define ENABLE_FEATURE 0 // Фича выключена
#ifdef ENABLE_FEATURE
runFeature(); // ВСЁ РАВНО ВЫПОЛНИТСЯ, так как макрос существует!
#endif

✅ Как исправить:

#if ENABLE_FEATURE
runFeature(); // Теперь сработает правильно (0 — ложь)
#endif

📝 Мини-тест для самопроверки

1. Во что превратится вызов SQUARE(2 + 2), если макрос объявлен как #define SQUARE(x) x * x?

  • А) 16
  • Б) 2 + 2 * 2 + 2 (что в Си равно 8)
  • В) (2 + 2) * (2 + 2)
  • Г) Ошибка компиляции

2. Какая директива препроцессора используется для полной остановки компиляции с выводом сообщения об ошибке?

  • А) #exit
  • Б) #warning
  • В) #error
  • Г) #pragma stop

3. Выполнится ли код внутри блока #ifdef, если макрос был объявлен как #define MY_MACRO 0?

  • А) Да, потому что макрос определен (существует).
  • Б) Нет, потому что его значение равно 0 (ложь).
  • В) Программа аварийно завершится во время компиляции.

4. Какую задачу выполняет оператор ## в макросах Си?

  • А) Превращает аргумент макроса в текстовую строку в кавычках.
  • Б) Склеивает два текстовых токена (идентификатора) в один.
  • В) Выполняет математическое деление по модулю.

<details>

  1. Б (Препроцессор подставит текст напрямую: 2 + 2 * 2 + 2. По правилам приоритетов операций сначала выполнится умножение 2 * 2 = 4, а затем сложение 2 + 4 + 2 = 8).
  2. В (#error).
  3. А (Директиве #ifdef абсолютно не важно значение, важен только сам факт создания макроса с помощью #define).
  4. Б (Это оператор конкатенации/склеивания токенов). </details>
