🧱 1. Базовые макросы (Объектоподобные)

Объектоподобные макросы выполняют простую и слепую замену текста в коде до начала компиляции. В языке Си они традиционно используются для объявления глобальных констант и конфигурационных параметров.

#define PI 3.14159
#define MAX_BUFFER_SIZE 1024
printf("Размер буфера: %d\n", MAX_BUFFER_SIZE);
#undef MAX_BUFFER_SIZE // ❌ С этого момента макрос больше недоступен

💡 Правила хорошего тона (Best Practices)

  • Именование: Всегда пишите имена макросов ИМЕНЕМ_В_ВЕРХНЕМ_РЕГИСТРЕ, чтобы визуально отличать их от обычных переменных в коде.
  • Изоляция области видимости: Если макрос нужен только внутри локального участка кода или конкретной функции, используйте директиву **#undef** сразу после завершения этого участка. Это предотвратит загрязнение глобальной области видимости и случайные совпадения имен.

⚙️ 2. Функциональные макросы (Функциоподобные)

Функциональные макросы способны принимать аргументы, имитируя работу обычных Си-функций. Однако, в отличие от функций, они не создают накладных расходов на вызов в стеке памяти, так как их код встраивается напрямую.

[!CAUTION] Золотое правило макросов: Всегда оборачивайте в круглые скобки как каждый отдельный аргумент, так и все выражение целиком!

🔍 Почему скобки жизненно необходимы?

Рассмотрим пример макроса возведения числа в квадрат:

// 🟢 ПРАВИЛЬНО: полная изоляция скобками
#define SQUARE(x) ((x) * (x))
// 🔴 НЕПРАВИЛЬНО: без скобок
#define BAD_SQUARE(x) x * x

Если вы вызовете BAD_SQUARE(5 + 1), препроцессор сделает текстовую замену: 5 + 1 * 5 + 1. По законам математики сначала выполнится умножение (1 * 5), и результат будет равен 11 вместо ожидаемых 36 (6²). Правильный макрос SQUARE(5 + 1) развернется в ((5 + 1) * (5 + 1)), что гарантирует математическую точность.

🛠️ 3. Операторы препроцессора (<tt>#</tt> и <tt>##</tt>)

Препроцессор Си имеет два уникальных оператора, которые позволяют творить магию генерации кода и метапрограммирования.

🧵 Оператор <tt>#</tt> — Взятие в кавычки (Stringification)

Превращает переданный аргумент в строковый литерал (заключает его в двойные кавычки "").

#define TO_STRING(x) #x
const char* name = TO_STRING(Hello); // Превратится в: "Hello"

🔗 Оператор <tt>##</tt> — Склейка токенов (Concatenation)

Склеивает два текстовых идентификатора во время препроцессинга, создавая новое имя переменной, функции или структуры.

#define MAKE_VAR(name, id) name##_##id
int MAKE_VAR(player, 1) = 100; // Создаст реальную переменную: int player_1 = 100;

🪐 4. Встроенные макросы языка

Компилятор Си автоматически предоставляет набор предопределенных макросов. Они незаменимы для трассировки кода, создания кастомных систем логирования и отладки критических ошибок (assert).

#include <stdio.h>
void logLocation(void) {
printf("Файл: %s\n", __FILE__);
printf("Строка: %d\n", __LINE__);
printf("Дата: %s\n", __DATE__);
printf("Время: %s\n", __TIME__);
}

📊 Справочник встроенных идентификаторов

Макрос Тип данных Что подставляет препроцессор?
__FILE__ const char* Полный путь или имя текущего исходного файла.
__LINE__ int Номер строки кода, на которой вызван макрос.
__DATE__ const char* Дата компиляции проекта в формате "Mmm Dd Yyyy".
__TIME__ const char* Точное время компиляции проекта в формате "Hh:Mm:Ss".

🔥 Пример продвинутого логирования ошибок

Объединив оператор # и встроенные макросы, можно создать красивый макрос отладки, который автоматически показывает, где и какое выражение пошло не так:

#define CHECK_CONDITION(cond) \
if (!(cond)) { \
printf("\({\color{#cf222e}\text{КРИТИЧЕСКАЯ ОШИБКА}}\) в файле %s (строка %d): Выражение (%s) неверно!\n", \
__FILE__, __LINE__, #cond); \
}
// Использование в коде:
CHECK_CONDITION(ptr != NULL);

🚀 Как увидеть раскрытые макросы?

Если вам интересно посмотреть, как препроцессор заменил ваши макросы на реальный текст, выполните команду сборки с флагом -E:

gcc -E main.c -o expanded_main.i

Откройте файл expanded_main.i — внутри него все ваши SQUARE, MAKE_VAR и __LINE__ уже будут заменены на итоговый Си-код.
