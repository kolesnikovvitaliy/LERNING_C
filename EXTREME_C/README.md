# ⚡ ЭКСТРЕМАЛЬНЫЙ СИ ⚡
## 🔥 Параллелизм, ООП и продвинутые возможности 🔥

## 🚀 О проекте

Добро пожаловать на территорию, где **C** раскрывается на полную мощность.

Этот репозиторий — ультимативный гид и база кода для тех, кто хочет выйти далеко за рамки классического синтаксиса. Здесь мы:
* Заставляем процедурный язык работать по правилам ООП.
* Выжимаем все соки из многопоточности.
* Погружаемся в темную магию компиляторов.

---

## 🛠 Ключевые направления

### 🧵 1. Экстремальный параллелизм
* **POSIX Threads (pthreads)**
* Управление жизненным циклом потоков.
* Настройка атрибутов потоков.
* **Синхронизация уровня Hardcore**
* Использование мьютексов и спин-локов.
* Работа с условными переменными и барьерами.
* **Атомарные операции**
* Применение библиотеки `<stdatomic.h>`.
* Проектирование Lock-free структур данных.
* **Параллельные паттерны**
* Реализация масштабируемых Thread Pool.
* Алгоритмы Work Stealing.
* Конвейерная обработка данных.

### 🏛 2. Объектно-Ориентированный Си (ООП без C++)
* **Инкапсуляция**
* Скрытие реализации через непрозрачные указатели (`opaque pointers`).
* Использование неполных типов (`incomplete types`).
* **Наследование**
* Эмуляция через ручное вложение структур.
* Контроль выравнивания памяти.
* **Полиморфизм**
* Динамическая диспетчеризация.
* Создание таблиц виртуальных методов (`vtable`).
* **Управление ресурсами**
* Реализация паттерна RAII.
* Написание кастомных деструкторов.

### 🔮 3. Продвинутые возможности и метапрограммирование
* **Магия препроцессора**
* Макросы с переменным числом аргументов (`__VA_ARGS__`).
* Автоматическая генерация кода через X-Macros.
* **Спецификаторы стандартов C11/C23**
* Статическая перегрузка функций через `_Generic`.
* Проверки на этапе компиляции через `_Static_assert`.
* **Работа с памятью**
* Написание кастомных аллокаторов.
* Использование арен памяти (`Memory Arenas`).
* Полное предотвращение утечек.
* **Инструменты отладки**
* Профилирование с помощью Valgrind.
* Статический анализ через AddressSanitizer (ASan).
* Поиск состояний гонки через ThreadSanitizer (TSan).
