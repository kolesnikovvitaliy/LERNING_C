# 🔀 Руководство по Си-Препроцессору: Условная Компиляция

Добро пожаловать в третью часть руководства по препроцессору Си! Этот раздел посвящен **условной компиляции** — механизму, который позволяет управлять тем, какие именно участки исходного кода будут скомпилированы, а какие препроцессор полностью сотрет. Это незаменимо для создания кроссплатформенных приложений и переключения режимов работы программы.

---

## ❓ 1. Базовые проверки (`#if`, `#else`, `#elif`, `#endif`)

Директивы условной компиляции работают аналогично обычным операторам `if-else` в Си, но с одним ключевым отличием: проверка происходит **до компиляции**. В итоговый исполняемый файл попадает только тот кусок кода, условие которого выполнилось (`true`).

> [!IMPORTANT]
> Весь неактивный код препроцессор физически удаляет из файла. Он не занимает ни одного байта в памяти готовой программы!

```c
#define ACCESS_LEVEL 2

#if ACCESS_LEVEL == 1
    void openGuestMenu(void);
#elif ACCESS_LEVEL == 2
    void openAdminMenu(void); // ✅ Препроцессор оставит только эту строку!
#else
    void openBlockMenu(void);
#endif
```

### 🧩 Как это выглядит для компилятора?
После этапа препроцессинга компилятор увидит этот участок кода в максимально усеченном виде:
```c
void openAdminMenu(void);
```

## 🔍 2. Проверка существования макросов (`#ifdef`, `#ifndef`)

Эти директивы проверяют сам факт наличия макроса в таблице препроцессора. Им **абсолютно неважно значение** макроса — даже если он объявлен как `#define DEBUG_MODE 0` или просто `#define DEBUG_MODE`, проверка вернет истину.

* **`#ifdef`** (IF DEFined) — выполнить код, если макрос **существует**.
* **`#ifndef`** (IF Not DEFined) — выполнить код, если макроса **не существует**.

```c
#define DEBUG_MODE

#ifdef DEBUG_MODE
    puts("Отладочный вывод: Переменная х = 5");
```

### 💡 Лайфхак: Быстрое комментирование кода через `#if 0`
Обычные комментарии `/* ... */` нельзя вкладывать друг в друга — это вызывает ошибку. Если вам нужно временно скрыть огромный кусок кода со своими комментариями или написать черновые заметки, используйте конструкцию `#if 0`:

```c
#if 0
    Этот код полностью проигнорируется компилятором.
    Сюда можно писать любые заметки, ТЕКСТ НА РУССКОМ или временно скрывать сломанный код.
    /* Внутренние комментарии больше ничего не сломают */
#endif
```

## 🌍 3. Практическое применение (Кроссплатформенность)

Условная компиляция — единственный способ заставить один и тот же исходный код успешно собираться под разные операционные системы (Windows, Linux, macOS). Компиляторы автоматически создают встроенные макросы (например, `_WIN32`, `__linux__`), на основе которых мы можем подключать специфичные системные библиотеки.

Для сложных проверок нескольких условий используется современный оператор `defined()`.

```c
#if defined(_WIN32)
    #include <windows.h>
    void clearScreen(void) { system("cls"); } // 🪟 Вызов консоли Windows
#elif defined(__linux__) || defined(__APPLE__)
    #include <unistd.h>
    void clearScreen(void) { system("clear"); } // 🐧🍏 Вызов терминала Unix/Mac
#endif
```

### 📊 Популярные системные макросы компиляторов

| Операционная система | Встроенный макрос | Назначение |
| :--- | :--- | :--- |
| **Windows (32/64-bit)** | `_WIN32` | ${\color{#0969da}\text{Автоматически задан в MSVC, MinGW}}$ |
| **Linux** | `__linux__` | ${\color{#2ea44f}\text{Автоматически задан в GCC, Clang на Linux}}$ |
| **macOS** | `__APPLE__` | ${\color{#magenta}\text{Автоматически задан в Apple Clang}}$ |

## 🚀 Полноценный пример: Логирование в зависимости от режима

Объединив условную компиляцию и макросы, можно создать гибкую систему логирования, которая автоматически отключается в релизной версии программы, повышая её скорость:

```c
#include <stdio.h>

// Раскомментируйте строку ниже для включения режима отладки
// #define ENABLE_DEBUG

#ifdef ENABLE_DEBUG
    #define LOG(msg) printf("\({\color{#d4a727}\text{[LOG]}}\) %s\n", msg)
#else
    #define LOG(msg) // 💨 В релизе макрос превращается в пустоту и ничего не делает
#endif

int main() {
    LOG("Запуск инициализации системы..."); // Вызовется только в DEBUG-режиме
    return 0;
}
```
