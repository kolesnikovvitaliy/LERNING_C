## 📌 Часть 1: Директивы препроцессора
*Узнайте, как подготавливать файлы и управлять процессом сборки.*

* **🗂️ Подключение файлов (`#include`)**
  * Разница между `<header.h>` (системные) и `"header.h"` (пользовательские).
  * Как препроцессор буквально «копирует и вставляет» текст файлов.
* **🛡️ Защита от повторного включения (Header Guards)**
  * Классический подход: `#ifndef`, `#define`, `#endif`.
  * Современная альтернатива: `#pragma once` (плюсы и минусы).
* **⚠️ Управление ошибками и предупреждениями**
  * Генерация ошибок компиляции через `#error`.
  * Вывод предупреждений с помощью `#warning`.
* **⚙️ Системные директивы**
  * Изменение нумерации строк через `#line`.
  * Использование `#pragma` для тонкой настройки компилятора.

## ⚡ Часть 2: Макросы
*Научитесь автоматизировать рутину и создавать гибкие подстановки.*

* **🧱 Базовые макросы (Объектоподобные)**
  * Определение констант через `#define`.
  * Правило хорошего тона: почему нужно удалять макросы через `#undef`.
* **⚙️ Функциональные макросы (Функциоподобные)**
  * Создание макросов с аргументами (например, `#define SQUARE(x) ((x) * (x))`).
  * **Важно:** Обязательное использование скобок для защиты приоритета операций.
  * Опасности побочных эффектов (почему `SQUARE(++i)` — это плохо).
* **🛠️ Операторы препроцессора**
  * Оператор `#` (Stringification) — превращение аргумента в строку.
  * Оператор `##` (Concatenation) — склеивание двух токенов в один.
* **🪐 Встроенные макросы языка**
  * Вывод информации о коде: `__FILE__`, `__LINE__`, `__DATE__`, `__TIME__`.

# 🚀 Полное руководство: Препроцессор в языке Си (C)
> **Изучаем директивы, макросы и условную компиляцию на практических примерах**

---

## 📌 Часть 1: Директивы препроцессора

### 🗂️ Подключение файлов (`#include`)
Препроцессор буквально копирует содержимое указанного файла в место вызова.
* `<header.h>` — поиск в системных каталогах (стандартная библиотека).
* `"header.h"` — поиск в текущей папке проекта (если не найден, то в системных).

```c
#include <stdio.h>   // Системный заголовочный файл для ввода-вывода
#include "my_math.h" // Наш собственный файл из папки проекта
```

### 🛡️ Защита от повторного включения (Header Guards)
Предотвращает ошибку двойного переопределения типов и структур, если один заголовочный файл случайно подключили несколько раз.

```c
// Способ 1: Классический и 100% переносимый (Стандарт ANSI C)
#ifndef MY_HEADER_H
#define MY_HEADER_H

void myFunction(void);

#endif // MY_HEADER_H

// Способ 2: Современный (поддерживается почти всеми компиляторами GCC, Clang, MSVC)
#pragma once
void myModernFunction(void);
```

### ⚠️ Управление ошибками и предупреждениями
Позволяет остановить компиляцию или вывести текстовое предупреждение прямо в терминал сборки.

```c
#if !defined(VERSION)
  #error "Критическая ошибка: VERSION не задана! Компиляция прервана."
#endif

#if defined(DEPRECATED_MODE)
  #warning "Внимание: Вы используете устаревший режим работы кода."
#endif
```

### ⚙️ Системные директивы (`#line` и `#pragma`)
* `#line` меняет внутренний счетчик строк компилятора (полезно для генераторов кода).
* `#pragma` задает специфичные команды конкретному компилятору.

```c
#line 100 "fake_file.c" // Теперь компилятор думает, что это 100-я строка файла fake_file.c

#pragma pack(push, 1) // Выравнивание структуры по 1 байту (для экономии памяти)
struct PackedData {
    char id;
    int value;
};
#pragma pack(pop) // Возвращаем стандартное выравнивание
```

## ⚡ Часть 2: Макросы

### 🧱 Базовые макросы (Объектоподобные)
Простая замена текста. В Си макросы часто заменяют константы. Хорошим тоном считается удалять макрос через `#undef`, если он больше не нужен.

```c
#define PI 3.14159
#define MAX_BUFFER_SIZE 1024

printf("Размер буфера: %d\n", MAX_BUFFER_SIZE);

#undef MAX_BUFFER_SIZE // Макрос больше недоступен ниже этой строки
```

### ⚙️ Функциональные макросы (Функциоподобные)
Макросы, которые принимают аргументы. Требуют строгой изоляции скобками.

```c
// ПРАВИЛЬНО: каждый аргумент и весь макрос обернуты в скобки
#define SQUARE(x) ((x) * (x))

int result = SQUARE(5); // Превратится в: ((5) * (5))
```

### 🛠️ Операторы препроцессора (`#` и `##`)
* `#` — оператор взятия в кавычки (Stringification). Превращает аргумент в строку.
* `##` — оператор склейки токенов (Concatenation). Склеивает два идентификатора в один.

```c
// Оператор #
#define TO_STRING(x) #x
const char* name = TO_STRING(Hello); // Превратится в "Hello"

// Оператор ##
#define MAKE_VAR(name, id) name##_##id
int MAKE_VAR(player, 1) = 100; // Создаст переменную: int player_1 = 100;
```

### 🪐 Встроенные макросы языка
Автоматически создаются компилятором. Незаменимы для написания логов (`printf`) и отладки ошибок.

```c
#include <stdio.h>

void logLocation(void) {
    printf("Файл: %s\n", __FILE__);
    printf("Строка: %d\n", __LINE__);
    printf("Дата: %s\n", __DATE__);
    printf("Время: %s\n", __TIME__);
}
```

## 🔀 Часть 3: Условная компиляция

### ❓ Базовые проверки (`#if`, `#else`, `#elif`, `#endif`)
В итоговый исполняемый файл попадет только тот кусок кода, условие которого выполнилось. Остальной код препроцессор сотрет.

```c
#define ACCESS_LEVEL 2

#if ACCESS_LEVEL == 1
    void openGuestMenu(void);
#elif ACCESS_LEVEL == 2
    void openAdminMenu(void); // Выберется этот блок
#else
    void openBlockMenu(void);
#endif
```

### 🔍 Проверка существования макросов (`#ifdef`, `#ifndef`)
Проверяют, был ли объявлен макрос вообще, независимо от его значения (даже если оно равно 0).

```c
#define DEBUG_MODE

#ifdef DEBUG_MODE
    puts("Отладочный вывод: Переменная х = 5");
#endif

// Быстрое комментирование кода:
#if 0
    Этот код полностью проигнорируется компилятором.
    Сюда можно писать любые заметки или временно скрывать сломанный код.
#endif
```

### 🌍 Практическое применение (Кроссплатформенность)
Один и тот же исходный код компилируется по-разному на разных операционных системах.

```c
#if defined(_WIN32)
    #include <windows.h>
    void clearScreen(void) { system("cls"); }
#elif defined(__linux__) || defined(__APPLE__)
    #include <unistd.h>
    void clearScreen(void) { system("clear"); }
#endif
```

## 🚨 Типичные ошибки новичков при работе с макросами

### 1. Отсутствие скобок в функциональных макросах
Самая частая логическая ошибка в Си, ломающая приоритет математических операций.
❌ **Плохо:**
```c
#define MULTIPLY(a, b) a * b
int x = MULTIPLY(2 + 3, 4); // Развернется в: 2 + 3 * 4 = 14 (Ожидалось: 20)
```
✅ **Как исправить:**
```c
#define MULTIPLY(a, b) ((a) * (b))
int x = MULTIPLY(2 + 3, 4); // Развернется в: ((2 + 3) * (4)) = 20
```

### 2. Побочные эффекты (Инкременты в аргументах)
Если передать в макрос переменную с инкрементом (`++`), операция выполнится несколько раз, так как текст подставляется напрямую.
❌ **Плохо:**
```c
#define SQUARE(x) ((x) * (x))
int i = 3;
int result = SQUARE(++i); // Развернется в: ((++i) * (++i)) -> i увеличится дважды!
```
✅ **Как исправить:** Для сложных вычислений в современном Си (начиная с C99) лучше использовать статические инлайновые функции: `static inline int square(int x) { return x * x; }`.

### 3. Лишняя точка с запятой `;` в конце `#define`
Препроцессор бездумно подставляет всё, что вы написали, включая точку с запятой, ломая синтаксис.
❌ **Плохо:**
```c
#define MAX_VALUE 100; // Обратите внимание на ';' в конце
int arr[MAX_VALUE];    // Развернется в ошибку: int arr[100;];
```
✅ **Как исправить:** Никогда не ставьте `;` в конце директив препроцессора.

### 4. Попытка проверить значение через `#ifdef` вместо `#if`
`#ifdef` проверяет только факт существования макроса, а не то, что внутри него.
❌ **Плохо:**
```c
#define ENABLE_FEATURE 0 // Фича выключена

#ifdef ENABLE_FEATURE
    runFeature(); // ВСЁ РАВНО ВЫПОЛНИТСЯ, так как макрос существует!
#endif
```
✅ **Как исправить:**
```c
#if ENABLE_FEATURE
    runFeature(); // Теперь сработает правильно (0 — ложь)
#endif
```
