# Комплект проверочных материалов по языку C

Данный репозиторий содержит комплект проверочных работ и тестов по программированию на языке C. Материалы выстроены по принципу от простого к сложному и ориентированы на проверку навыков работы с препроцессором, указателями на функции, безопасностью памяти и оптимизацией архитектуры.

---

## 📅 Проверочная работа №1: Макросы, препроцессор и безопасность

**Цель:** Проверить понимание работы препроцессора (`##`, `#`), развёртывания макросов и базовой безопасности при работе со строками.

### Задание (Практика)
1. Выполните ручное развёртывание макроса: во что превратится строка `CMD(copy);` после этапа препроцессинга? Напишите итоговый C-код.
2. Объясните назначение операторов `#` и `##` в директиве `#define CMD(NAME)`.
3. Найдите уязвимость в строке `scanf("%s", cmd);`. Напишите исправленный вариант, предотвращающий переполнение буфера (Buffer Overflow).

### Тест №1
1. **Что делает оператор `##` в макросе `NAME ## _cmd`?**
   - А) Сравнивает две строки на равенство.
   - Б) Конкатенирует (склеивает) два токена на этапе препроцессинга в один идентификатор.
   - В) Находит остаток от деления строк.
   - Г) Создает указатель на переменную.

2. **Какое значение возвращает функция `strcmp(str1, str2)`, если строки полностью идентичны?**
   - А) `1`
   - Б) `-1`
   - В) `0`
   - Г) Любое положительное число.

3. **К какому сдвигу в памяти (в байтах) приведет запись строки из 10 символов через небезопасный `scanf("%s", cmd);`, если размер `cmd` равен 5 байтам?**
   - А) Ошибки не будет, буфер автоматически расширится.
   - Б) Произойдет запись 6 байт (включая `\0`) за пределы массива (переполнение стека).
   - В) Функция запишет только первые 4 символа и добавит `\0`.
   - Г) Программа штатно завершится с кодом `0`.

## 🔀 Проверочная работа №2: Указатели на функции и таблицы переходов (Jump Tables)

**Цель:** Научиться заменять громоздкие цепочки `if-else` массивами структур и указателями на функции для создания масштабируемой архитектуры.

### Задание (Практика)
Вместо последовательной проверки условий `if(strcmp(...) == 0)` перепишите логику интерпретатора:
1. Создайте функции для обработки каждой команды со следующей сигнатурой: `void do_copy(void);`, `void do_paste(void);`, `void do_cut(void);`.
2. Создайте структуру `Command`, содержащую имя команды и указатель на соответствующую функцию.
3. Оформите таблицу команд как массив таких структур. Реализуйте поиск и вызов функций в цикле. Если команда не найдена, выведите `Unknown command`.

### Тест №2
1. **Каков правильный синтаксис объявления указателя на функцию, которая ничего не принимает и ничего не возвращает (`void`)?**
   - А) `void *func_ptr(void);`
   - Б) `void (*func_ptr)(void);`
   - В) `void func_ptr*(void);`
   - Г) `int *func_ptr(void);`

2. **В чем главное архитектурное преимущество использования массива структур с указателями на функции вместо цепочки `if-else`?**
   - А) Код начинает занимать меньше места на жестком диске.
   - Б) Увеличивается скорость компиляции программы.
   - В) Повышается масштабируемость: для добавления новой команды достаточно добавить элемент в массив, не меняя логику выбора.
   - Г) Программа полностью защищается от утечек памяти.

3. **Что делает выражение `commands[i].handler();`, если `handler` — это указатель на функцию?**
   - А) Вызывает функцию, на которую ссылается указатель.
   - Б) Возвращает размер функции в байтах.
   - В) Проверяет, существует ли функция с таким именем.
   - Г) Удаляет функцию из памяти.

## 💾 Проверочная работа №3: Оптимизация памяти и аргументы CLI

**Цель:** Проверить навыки работы с константными строками, оптимизации использования стека и интеграции интерактивного режима с аргументами командной строки.

### Задание (Практика)
1. Избавьтесь от выделения фиксированных 256 байт под каждую строку команды на этапе инициализации (`char NAME##_cmd`). Используйте указатели на строковые литералы (`const char *`).
2. Модифицируйте функцию `main`. Если пользователь передал аргумент при запуске программы (например, `./program copy`), программа должна сразу выполнить эту команду без интерактивного ввода через `scanf`. Если аргументов нет — запускается стандартный интерактивный режим.

### Тест №3
1. **Какое объявление массива строк является наиболее эффективным по памяти для хранения фиксированных имен команд?**
   - А) `char commands = {"copy", "paste", "cut"};`
   - Б) `const char *commands[] = {"copy", "paste", "cut"};`
   - В) `int commands[] = {"copy", "paste", "cut"};`
   - Г) `char **commands = "copy paste cut";`

2. **Чему равно значение `argc` при запуске программы командой `./interpreter copy`?**
   - А) `0`
   - Б) `1`
   - В) `2`
   - Г) `3`

3. **Где в памяти будут храниться строки `"copy"`, `"paste"`, `"cut"`, если они объявлены как `const char *cmd = "copy";`?**
   - А) В стеке функции (Stack).
   - Б) В динамической памяти (Heap).
   - В) В секции только для чтения (Read-Only Data / Текстовый сегмент).
   - Г) В регистрах процессора.

## 🔑 Ответы и разборы тестов

### Разбор Теста №1
1. **Правильный ответ: Б.** Оператор `##` является оператором конкатенации токенов (склеивания). Он соединяет имя, переданное в макрос, с суффиксом `_cmd`.
2. **Правильный ответ: В.** Согласно стандарту языка C, функция `strcmp` возвращает `0`, если сравниваемые строки посимвольно равны и имеют одинаковую длину.
3. **Правильный ответ: Б.** Так как размер буфера 5 байт, а вводимая строка с завершающим нулем занимает 11 байт, лишние 6 байт запишутся в соседние ячейки памяти стека, разрушая данные (включая адрес возврата).

### Разбор Теста №2
1. **Правильный ответ: Б.** Круглые скобки вокруг `*func_ptr` обязательны. Вариант А (`void *func_ptr(void);`) объявляет обычную функцию, возвращающую нетипизированный указатель `void*`.
2. **Правильный ответ: В.** Использование таблиц переходов реализует принцип открытости/закрытости (OCP). Вы можете расширять систему (добавлять команды), декларативно дополняя массив, не изменяя императивный код поиска.
3. **Правильный ответ: А.** Синтаксис `идентификатор()` применяется к указателю на функцию так же, как и к обычному имени функции для ее вызова.

### Разбор Теста №3
1. **Правильный ответ: Б.** Массив указателей хранит лишь адреса (по 4 или 8 байт на указатель), ссылающиеся на строковые литералы ровно той длины, которая необходима. Вариант А из исходного кода тратил бы $3 \times 256 = 768$ байт.
2. **Правильный ответ: В.** `argc` считает общее количество аргументов, включая само имя запускаемого исполняемого файла. `./interpreter` — первый аргумент, `copy` — второй. Всего 2.
3. **Правильный ответ: В.** Строковые литералы размещаются в неизменяемом сегменте данных (обычно `.rodata`). Попытка изменить их через указатель вызовет Segment Fault.

## 🛠️ Эталонные решения практических заданий

### Решение работы №1 (Развёртывание макроса и scanf)
1. **Развёрнутый макрос `CMD(copy);`:**
   ```c
   char copy_cmd[256] = "";
   strcpy(copy_cmd, "copy");
   ```
2. **Объяснение знаков:** `#NAME` превращает переданный токен в строковый литерал (добавляет кавычки: `copy` -> `"copy"`). `NAME ## _cmd` склеивает токен со словом `_cmd` для создания имени переменной (`copy` -> `copy_cmd`).
3. **Безопасный ввод:** Чтобы избежать переполнения буфера `cmd`, необходимо ограничить ввод до 255 символов (оставив 1 байт под нуль-терминатор `\0`):
   ```c
   scanf("%255s", cmd);
   ```

### Комплексное эталонное решение (Работы №2 и №3 вместе)
Данный production-ready код объединяет использование **указателей на функции**, **таблицу команд**, **оптимизацию памяти** и **обработку аргументов командной строки**.

```c
#include <stdio.h>
#include <string.h>

// 1. Определение функций обработки команд
void do_copy(void) {
    printf("Логика: Данные скопированы в буфер обмена.\n");
}

void do_paste(void) {
    printf("Логика: Данные вставлены из буфера обмена.\n");
}

void do_cut(void) {
    printf("Логика: Данные вырезаны и сохранены в буфер.\n");
}

// 2. Определение структуры таблицы переходов
typedef struct {
    const char *name;      // Имя команды (экономим память стека)
    void (*handler)(void); // Указатель на функцию-обработчик
} Command;

// 3. Инициализация таблицы команд
const Command command_table[] = {
    {"copy",  do_copy},
    {"paste", do_paste},
    {"cut",   do_cut}
};

// Вычисляем количество команд динамически на этапе компиляции
const size_t num_commands = sizeof(command_table) / sizeof(command_table[0]);

// 4. Основной движок интерпретатора
void execute_command(const char *cmd) {
    for (size_t i = 0; i < num_commands; i++) {
        if (strcmp(cmd, command_table[i].name) == 0) {
            command_table[i].handler(); // Вызов функции по указателю
            return;
        }
    }
    printf("Error: Unknown command '%s'\n", cmd);
}

int main(int argc, char **argv) {
    // Если передан аргумент командной строки (argc > 1), обрабатываем его сразу
    if (argc > 1) {
        printf("Режим CLI. Выполнение команды: %s\n", argv[1]);
        execute_command(argv[1]);
        return 0;
    }

    // Иначе запускаем интерактивный режим
    printf("Интерактивный режим. Введите команду (copy/paste/cut):\n");
    char cmd[256];

    // Безопасный ввод с ограничением по размеру буфера
    if (scanf("%255s", cmd) == 1) {
        execute_command(cmd);
    }

    return 0;
}
```
