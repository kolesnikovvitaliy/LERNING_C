# 📝 Проверочные работы и тесты: Имитация циклов в макросах Си

Данный методический материал содержит **3 проверочные работы** разного уровня сложности и **тестовые задания** к ним для проверки понимания механизмов раскрытия вариативных макросов (`__VA_ARGS__`), токенизации и каскадных конвейеров на этапе препроцессинга.

---

## 🛠️ Проверочная работа №1: Базовый уровень
**Тема**: Каскадное раскрытие и строковое преобразование (`#X`)

### 📋 Задание
Изучите приведенный ниже код и выполните задания:
```c
#define STEP_2(X, ...) printf("[%s]", #X);
#define STEP_1(X, ...) printf("[%s]", #X); STEP_2(__VA_ARGS__)
#define RUN(...) STEP_1(__VA_ARGS__)

int main(void) {
    RUN(alpha, beta, gamma);
    return 0;
}
```
1. Напишите, какой текст будет выведен на экран после компиляции и запуска программы.
2. Объясните, что произойдет с токеном `gamma` на этапе работы препроцессора.
3. Во что развернется вызов `RUN(one two three)`?

### ✏️ Тест к проверочной работе №1
1. **Какая строка кода будет сгенерирована препроцессором для вызова `RUN(a, b)`?**
   * А) `printf("[%s]", "a"); printf("[%s]", "b");`
   * Б) `printf("[%s]", "a"); STEP_2(b);`
   * В) `printf("[%s]", "a"); printf("[%s]", "b", __VA_ARGS__);`

2. **Что делает оператор `#` перед именем макро-аргумента (например, `#X`)?**
   * А) Превращает переданный токен в строковый литерал на этапе компиляции.
   * Б) Обозначает числовой тип данных.
   * В) Вызывает рекурсивный обход аргументов.

3. **Что произойдет, если вызвать макрос `RUN()` без аргументов на старом компиляторе (до стандарта C23)?**
   * А) Программа успешно выведет пустую строку.
   * Б) Произойдет ошибка препроцессинга из-за синтаксиса "висящей запятой".
   * В) Макрос уйдет в бесконечный рантайм-цикл.

## 🚀 Проверочная работа №2: Средний уровень
**Тема**: Автоматический подсчет аргументов и склейка токенов (`##`)

### 📋 Задание
Перед вами код системы безопасного освобождения памяти:
```c
#define _COUNT_INTERNAL(_1, _2, _3, _4, N, ...) N
#define ARGS_COUNT(...) _COUNT_INTERNAL(__VA_ARGS__, 4, 3, 2, 1, 0)

#define CONCAT_INTERNAL(A, B) A ## B
#define CONCAT(A, B) CONCAT_INTERNAL(A, B)

#define PROCESS_1(X)      free(X);
#define PROCESS_2(X, ...) free(X); PROCESS_1(__VA_ARGS__)

#define SMART_FREE(...) CONCAT(PROCESS_, ARGS_COUNT(__VA_ARGS__))(__VA_ARGS__)
```
1. Рапишите пошагово, как препроцессор вычисляет значение `ARGS_COUNT(ptr1, ptr2)`. Какие именно позиции занимают токены?
2. Напишите конечный вид макроса, который сформирует функция `CONCAT` для вызова `SMART_FREE(ptr1, ptr2)`.
3. Какое максимальное количество указателей способен обработать данный каскад макросов без внесения изменений в код?

### ✏️ Тест к проверочной работе №2
1. **Какое значение вернет макрос `ARGS_COUNT(data1, data2, data3)`?**
   * А) 4
   * Б) 3
   * В) 2

2. **Для чего в макросах используется оператор склейки токенов `##`?**
   * А) Для конкатенации строк в рантайме.
   * Б) Для объединения двух лексем в один программный токен на этапе препроцессинга.
   * В) Для разделения элементов массива.

3. **Почему для склейки используются два макроса (`CONCAT` и `CONCAT_INTERNAL`), а не один?**
   * А) Чтобы разрешить препроцессору сначала раскрыть внутренние макросы-аргументы (например, `ARGS_COUNT`), а уже потом склеить результат.
   * Б) Это требование безопасности для предотвращения утечек памяти.
   * В) Один макрос не умеет работать со строками.

## 💎 Проверочная работа №3: Профессиональный уровень
**Тема**: Изоляция побочных эффектов (`do-while(0)`) и защита от аномалий

### 📋 Задание
Проанализируйте два варианта реализации атомарного действия внутри макро-цикла:
```c
// Вариант А
#define SAFE_FREE_A(ptr) free(ptr); ptr = NULL;

// Вариант Б
#define SAFE_FREE_B(ptr) do { free(ptr); (ptr) = NULL; } while(0)
```
1. Докажите на примере конструкции `if (condition) SAFE_FREE_A(p); else ...`, почему Вариант А приводит к критической синтаксической ошибке или нарушению логики ветвления.
2. Почему в Варианте Б аргумент `(ptr)` внутри тела макроса обернут в круглые скобки? К каким потенциальным проблемам может привести отсутствие скобок при передаче сложных выражений?
3. Модифицируйте Вариант Б так, чтобы макрос перед освобождением памяти дополнительно выводил имя освобождаемого указателя на экран.

### ✏️ Тест к проверочной работе №3
1. **Какую основную задачу решает обертка `do { ... } while(0)` в макросах?**
   * А) Создает реальный рантайм-цикл, который оптимизирует компилятор.
   * Б) Превращает группу инструкций в единый безопасный блок (составной оператор), который ведет себя как обычная функция и корректно работает внутри `if-else`.
   * В) Ускоряет фазу препроцессинга файла.

2. **Что произойдет, если вызвать `FREE_ALL(a, b, c, d, e)` в каскаде, рассчитанном максимум на 4 элемента?**
   * А) Компилятор выдаст ошибку `Segmentation Fault`.
   * Б) Элемент `e` будет молча проигнорирован терминальным макросом, и память под ним не очистится.
   * В) Препроцессор начнет бесконечную генерацию кода.

3. **Какая современная директива (начиная с C++20/C23) позволяет полностью отказаться от создания жестких каскадов типа `LOOP_1, LOOP_2` для реализации циклов?**
   * А) `__VA_ARGS__`
   * Б) `__VA_OPT__`
   * В) `#pragma once`
