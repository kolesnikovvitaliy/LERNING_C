* #  Использование макросов для генерации цикла
Практическое применение препроцессора Си для создания кастомных синтаксических конструкций. На примере связки макросов `LOOP` и `ENDLOOP` продемонстрирована генерация стандартного цикла `for` со строгим инклюзивным шагом. Особое внимание уделено инкапсуляции и ограничению области видимости переменной-счетчика внутренним блоком итерации.

```c
// Пример использования в коде:
LOOP(counter, 1, 10)
    PRINT(counter)
ENDLOOP
```
### Развертывание макросов препроцессором

При выполнении команды препроцессинга `gcc -E main.c` все макроопределения разворачиваются в чистый Си-код непосредственно перед компиляцией:

```c
// Результат работы препроцессора:
int main(int args, char **argv) {
    for (int counter = 1; counter <= 10; counter++) {
        printf("%d\n", counter);
    }
    return 0;
}
```

> ⚠️ **Важно**: Переменная `counter` инициализируется внутри выражения `for`. Её область видимости ограничена данным циклом, и она автоматически уничтожается после выхода из блока.

## Обзор разделов

### Определение макроса (`#define`)
Изучение синтаксиса директивы `#define` для создания именованных констант. Рассматривается процесс инициализации макроса `ABC` со значением `5`. Данный подход позволяет избавиться от «магических чисел» в коде, улучшает читаемость и упрощает дальнейшую поддержку и конфигурирование программы.

### Подстановка макроса в функции `main`
Практический пример развертывания макроса внутри исполняемого кода. Препроцессор автоматически заменяет маркер `ABC` на целочисленный литерал `5` в месте инициализации переменной `y`. Документирование строки выполнено с помощью специального тега Doxygen (`/**< ... */`), предназначенного для описания элементов кода в той же строке.

```c
#define ABC 5

int demo_macro_substitution_in_main(int argc, char **argv)
{
    int x = 2;
    int y = ABC; /**< Здесь препроцессор заменит ABC на 5 */

    int z = x + y;
    return 0;
}
```

### Определение функционального макроса
Анализ работы параметризированных макросов, принимающих аргументы. На примере вызова `ADD(x, y)` демонстрируется, как препроцессор на этапе компиляции разворачивает выражение в безопасную арифметическую форму `((x) + (y))`. Это исключает накладные расходы на вызов полноценной функции и оптимизирует вычисления в стеке программы.

### Документирование локального контекста
Разбор внутренней архитектуры функции `main` со встроенной спецификацией Doxygen (`@brief`, форматирование ключевых слов оператором `\c`). Раздел иллюстрирует правила работы с локальными переменными, подавление предупреждений компилятора о неиспользуемом коде с помощью приведения к `(void)`, а также обеспечение строгого соответствия типов возвращаемого значения сигнатуре функции.

```c
#define ADD(a, b) ((a) + (b))
int main(int argc, char **argv) {
    /**
     * @brief Локальная переменная 'x'.
     * Первое целое число (int), выступающее фундаментом нашего вычисления.
     * Инициализировано строгим значением 2. Occupies 4 bytes of memory standardly.
     */
    int x = 2;

    /**
     * @brief Локальная переменная 'y'.
     * Второе целое число (int), призванное дополнить первое.
     * Намертво связано с константой 5 в стеке программы.
     */
    int y = 5;

    /**
     * @brief Локальная переменная 'z'.
     * Священный грааль нашей программы, куда записывается результат сложения.
     * На этапе компиляции строка превратится в \c int \c z \c = \c ((x) \c + \c (y));
     * В текущих реалиях здесь родится и будет храниться число 7.
     */
    int z = ADD(x, y);

    /* Подавляем стенания компилятора о неиспользуемых системных переменных */
    (void)argc;
    (void)argv;
    (void)z;

    /**
     * @brief Финальный аккорд.
     * Мы осознанно заменили некорректный суффикс '0l' (long) на чистый '0',
     * чтобы тип возвращаемого значения идеально соответствовал сигнатуре int main.
     * Возвращаем ноль, как символ безупречной победы алгоритма над кремнием.
     */
    return 0;
}
```
