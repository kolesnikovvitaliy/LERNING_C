# 🛠️ Руководство по Си-Препроцессору: Директивы и Оптимизация

Добро пожаловать в подробное руководство по работе с препроцессором Си! Препроцессор — это мощный инструмент, который анализирует и модифицирует ваш исходный код **до** того, как начнется фактическая компиляция.

---

## 🗂️ 1. Подключение файлов (`#include`)

Директива `#include` дает команду препроцессору буквально скопировать все содержимое указанного файла и вставить его непосредственно в место вызова.

### 🔍 Различия в типах синтаксиса

```c
#include <stdio.h>   // 🌐 Системные каталоги
#include "my_math.h" // 📁 Локальная папка проекта
```

| Синтаксис | Где компилятор ищет файл? | Основное назначение |
| :--- | :--- | :--- |
| `<header.h>` | Ищет **только** в системных путях среды разработки (стандартная библиотека). | Подключение встроенных библиотек (`stdio.h`, `stdlib.h`, `math.h`). |
| `"header.h"` | Сначала ищет в **текущей директории** проекта. Если не находит, переходит в системные. | Подключение собственных заголовочных файлов проекта. |

---

## 🛡️ 2. Защита от повторного включения (Header Guards)

Если один и тот же заголовочный файл будет случайно подключен к файлу `.c` несколько раз (например, через цепочку других библиотек), это вызовет критическую ошибку компиляции из-за **повторного объявления** типов, макросов или структур. Для защиты используются специальные механизмы.

### 🏛️ Способ 1: Классический (Стандарт ANSI C)
Использует классические макросы условной компиляции. Этот метод имеет **100% переносимость** и гарантированно работает на абсолютно любых, даже самых старых компиляторах.

```c
#ifndef MY_HEADER_H
#define MY_HEADER_H

// 📦 Здесь объявляются ваши функции и структуры
void myFunction(void);

#endif // MY_HEADER_H
```

### ⚡ Способ 2: Современный подход
Более лаконичный и быстрый в написании метод. Он поддерживается всеми современными компиляторами (GCC, Clang, MSVC).

```c
#pragma once

// 📦 Содержимое файла защищено автоматически
void myModernFunction(void);
```

> [!NOTE]
> Использование `#pragma once` может слегка ускорить сборку больших проектов, так как компилятор даже не будет открывать файл повторно, в то время как классический `#ifndef` заставляет препроцессор заново считывать файл до самого конца.

## ⚠️ 3. Управление ошибками и предупреждениями

Вы можете управлять ходом сборки программы, заставляя компилятор выводить предупреждения или полностью прерывать процесс, если внешние условия или настройки проекта не соответствуют требованиям.

```c
#if !defined(VERSION)
  #error "Критическая ошибка: VERSION не задана! Компиляция прервана."
#endif

#if defined(DEPRECATED_MODE)
  #warning "Внимание: Вы используете устаревший режим работы кода."
#endif
```

* **`#error`** — генерирует критическую ошибку сборки. Компиляция кода **${\color{#cf222e}\text{полностью прекращается}}$**. Идеально для проверки обязательных настроек.
* **`#warning`** — выводит текстовое предупреждение в консоль терминала, но **${\color{#2ea44f}\text{позволяет сборке успешно завершиться}}$**. Полезно для маркировки устаревшего кода (Deprecated).

## ⚙️ 4. Системные директивы (`#line` и `#pragma`)

Эти директивы используются для тонкой настройки компилятора и низкоуровневого управления структурами данных.

### 📝 Подмена строк (`#line`)
Директива жестко изменяет внутренний счетчик строк и имя файла в компиляторе. Это критически важно при написании парсеров, лексеров или генераторов кода, чтобы отчеты об ошибках указывали на исходный шаблон, а не на сгенерированный код.

```c
#line 100 "fake_file.c"
// С этой секунды компилятор считает, что этот код находится на 100-й строке файла fake_file.c
```

### 🗜️ Упаковка структур (`#pragma pack`)
По умолчанию компилятор выравнивает данные в памяти по границам слов (например, по 4 или 8 байт) для ускорения доступа процессора к памяти. Это создает «дыры» (пустые байты) внутри структур. С помощью `#pragma pack` можно заставить компилятор упаковать данные максимально плотно.

```c
#pragma pack(push, 1) // 🧲 Включаем выравнивание строго по 1 байту

struct PackedData {
    char id;    // 1 байт
    int value;  // 4 байта
}; // Итоговый размер в памяти: ровно 5 байт (вместо стандартных 8 байт)

#pragma pack(pop) // ↩️ Сбрасываем настройки до стандартных значений
```

> [!CAUTION]
> Упаковка структур экономит оперативную память и незаменима при передаче пакетов по сети или записи во Flash-память микроконтроллера. Однако помните, что чтение невыровненных данных может **<u>снизить производительность</u>** процессора на некоторых архитектурах.

## 🚀 Как запустить только препроцессор?

Если вы хотите увидеть, во что превращается ваш код после работы препроцессора (как раскрываются макросы и вставляются файлы), выполните в терминале команду:

```bash
gcc -E main.c -o preprocessed_main.i
```
После этого откройте файл `preprocessed_main.i` — там вы увидите чистый Си-код, готовый к компиляции.
